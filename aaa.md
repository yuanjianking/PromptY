# 一、第一优先级（必须纳入，决定成败）

## 1️⃣ 目标模型（必选项）

你说得完全对：**Prompt 离开模型是不存在“最优解”的**。

### 影响点

* 指令遵循能力差异
* 对结构化输出的服从程度
* 对隐含上下文的“脑补”倾向
* 对角色设定的敏感度

### 产品中必须做的

* 强制用户选择或确认目标模型（或模型类型）
* Prompt 构造时**显式针对模型风格调整**
* 风险说明中必须写：

  > 「该 Prompt 针对 ×× 类模型优化」

📌 **这是你和“通用 Prompt 生成器”的本质区别**

---

## 2️⃣ 使用场景类型（必选项）

同一句 Prompt，在不同**任务类型**下是完全不同的。

### 常见场景类型

* 分析 / 推理
* 创作 / 发散
* 总结 / 压缩
* 判断 / 评估
* 生成结构化结果

### 产品动作

* 将“使用场景”强制结构化为任务类型
* Prompt 构造时锁定一种**主任务范式**
* 风险提示：

  > 「该 Prompt 不适合 ×× 类型任务」

---

## 3️⃣ 输出使用对象（极易被忽略，但非常重要）

### 影响点

* 输出语言风格
* 专业深度
* 冗余容忍度

### 示例

* 给专家看的
* 给决策者看的
* 给普通用户看的
* 给程序处理的（JSON / 表格）

📌 **这是输出“可用 vs 不可用”的分界线**

---

# 二、第二优先级（决定稳定性与可控性）

## 4️⃣ 上下文可获得性（关键）

Prompt 是否**依赖外部隐含信息**？

### 情况

* 强依赖背景（容易跑偏）
* 弱依赖背景（更稳定）

### 产品中要做的

* 判断是否存在「隐含前提」
* 明确告知用户：

  > 「当前 Prompt 假设模型知道 ×× 信息」

---

## 5️⃣ 输出确定性需求（必须显式）

这是 Prompt 工程的核心轴之一。

### 光谱

* 探索型（允许多答案）
* 平衡型
* 收敛型（强约束）

### 产品行为

* 强制选择或推断一个位置
* 风险提示：

  * 探索型 → 不可预测
  * 收敛型 → 容易遗漏

---

## 6️⃣ 失败容忍度（专业工具必备）

### 关键问题

> 输出“错一点”可不可以？

### 影响

* 是否允许模型假设
* 是否允许模型补全缺失信息
* 是否允许不确定回答

📌 **这直接决定 Prompt 的语气和边界**

---

# 三、第三优先级（影响体验与可迁移性）

## 7️⃣ Prompt 生命周期（一次性 vs 可复用）

### 区别

* 一次性 Prompt：可以激进
* 可复用 Prompt：必须保守、可控

### 产品中

* 明确标注 Prompt 类型
* 给出维护成本提示

---

## 8️⃣ 运行环境假设（弱相关，但要声明）

你无法控制，但必须提醒：

* 温度、top_p 不确定
* 是否启用系统 Prompt 不确定
* 是否存在上下文污染

👉 **你不控制，但你要告知风险**

---

## 9️⃣ 用户 Prompt 熟练度（隐性因素）

### 高手 vs 新手

* 新手更需要：

  * 明确结构
  * 强提示
* 老手更需要：

  * 灵活度
  * 可裁剪

📌 可作为产品个性化参数（后期）
